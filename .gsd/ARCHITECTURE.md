## Overview

## Data Flow

1. **Authentication**: Users sign in via Google OAuth. Frontend sends the credential to `/api/auth/google`. Backend verifies the JWT, creates/fetches the company in PostgreSQL, and establishes a session (locally via `localStorage` on frontend, validated via `x-api-key` header on subsequent requests).
2. **Test Lifecycle**: Company creates a test (`POST /api/tests`), uploading an APK. Backend stores the APK in B2/Supabase, records the test in DB. Admin approves the test (`PUT /api/admin/tests/:testId/status`). Testers claim the test (from Android App), submit bugs (videos/screenshots).
3. **AI Analysis**: Once bugs are submitted, backend invokes `analyzeBugReport` (Gemini API) to flag duplicate or invalid reports.
4. **Data Retrieval**: Frontend fetches approved tests, metrics, and bug reports through `/api/tests/*` and `/api/bugs/*`. Videos are proxied through `/api/videos/*` for security.
